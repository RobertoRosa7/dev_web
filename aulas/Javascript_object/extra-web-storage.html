Web Storage

Local Storage | Session Storage | IndexedDB | Web SQL | Cookies

No HTML5, o browser consegue guardar essas informações no lado do cliente, assim poderá fazer toda uma lógica quando esses mesmos dados forem verificado pelo servidor.

Local e Session Storage

Neste tipo de caso, as informações ficam armazenadas no browser de forma análoga ao banco de dados
Suportando uma quantidade maior de Cookies, no Session Storage os dados ficam armazenado no browser enquanto o browser não é reiniciado, quando for reiniciado a sessão será terminada. No Local Storage é um armazenamento persistente, enquando Session Storage é temporário.

IndexDB e Web SQL

São formas mais avançadas de armazenamento de dados no Browser, em termos de criação de tabelas e fazer consultas nos dados.

Cookies

São chaves que armazena informações relacionada a sessão, com o uso do cookies, as senhas e logins do usuário ficam armazenadas no browser assim, não será necessário preencher formulários sempre que acessar o site que precise de autenticação.
