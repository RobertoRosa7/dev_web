Pilar da Herança

Reutilizável

Significa que podemos usar o mesmo código em diversos outros projetos, sendo reutilizado, isto é, cria o código para um determinado projeto, e reutiliza em outro, podendo ser feito manutenção dele, para que se aplique em diversos outros, portanto, podemos criar um código mestre ou geral e a partir dele criar novos outro código baseado no geral.

Manutenção

Com a manutenção dos códigos, podemos reduzir as redundâncias e simplificar ainda mais os códigos, assim para um objeto que é genérico transformá-lo em um objeto específico utilizando o pilar da herança.
Para a manutenção ser eficaz, basta adicionar, modificar ou remover qualquer atributo ou método da classe pai (super-classe) para que todas as classes que estendem receber suas alterações.
